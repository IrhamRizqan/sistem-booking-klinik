# Prompting Instructions

This document defines how the AI should behave while working on this project.

---

# Before Coding

Always understand the existing architecture first.

Read every relevant file before making changes.

Never assume implementation details.

Explain the implementation plan before writing code.

Mention which files will be modified.

---

# During Coding

Keep changes as small as possible.

Do not rewrite unrelated files.

Do not rename files without permission.

Preserve project structure.

Follow existing coding style.

Respect MVC architecture.

Respect all files inside the `.ai` directory.

---

# Decision Making

When multiple implementations are possible:

Choose the simplest.

Choose the easiest to maintain.

Choose the easiest to explain during presentation.

Avoid unnecessary dependencies.

Avoid unnecessary packages.

---

# UI Generation

Follow Hallmark principles.

Use Bootstrap 5 only.

Avoid Bootstrap default appearance.

Avoid AI-generated dashboard aesthetics.

Maintain consistent spacing.

Maintain visual hierarchy.

Keep components reusable.

---

# Code Quality

Generate production-quality code.

Do not generate placeholder implementations.

Avoid TODO comments.

Avoid incomplete features.

Always finish one feature before moving to another.

---

# Communication

Before making changes:

1. Explain the plan.
2. List affected files.
3. Explain why the implementation was chosen.

After making changes:

1. Summarize what was implemented.
2. Mention any limitations.
3. Suggest the next logical step.

---

# Never

Never introduce new frameworks.

Never redesign the architecture.

Never change the database schema without approval.

Never remove existing functionality.

Never ignore project documentation.

Never over-engineer the solution.
